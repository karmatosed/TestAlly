# Glossary

A quick reference guide to the key tools and concepts used in TestAlly's technical documentation.

| Tool | Description |
|------|-------------|
| [axe-core](#axe-core) | Automated accessibility testing engine |
| [CodeMirror](#codemirror) | Extensible code editor component |
| [concurrently](#concurrently) | Run multiple npm scripts in parallel |
| [css-tree](#css-tree) | CSS parser and toolkit |
| [ESLint](#eslint) | Static code analysis for JavaScript/TypeScript |
| [htmlparser2](#htmlparser2) | Fast HTML parser |
| [jsdom](#jsdom) | JavaScript implementation of web standards |
| [LangChain.js](#langchainjs) | Framework for building LLM-powered applications |
| [Prettier](#prettier) | Opinionated code formatter |
| [Vite](#vite) | Modern frontend build tool |
| [Zod](#zod) | TypeScript-first schema validation |

---

## axe-core

**[axe-core](https://github.com/dequelabs/axe-core)** is an accessibility testing engine developed by Deque Systems. It runs automated WCAG 2.0+ compliance checks against HTML markup and returns detailed violation reports with remediation guidance.

---

## CodeMirror

**[CodeMirror](https://codemirror.net/)** (version 6) is a versatile code editor component library for the web. It provides syntax highlighting, line numbers, autocompletion, and extensive customization through a modular extension system.

---

## concurrently

**[concurrently](https://github.com/open-cli-tools/concurrently)** is a utility for running multiple npm scripts simultaneously with a single command. It's commonly used in monorepos to start multiple development servers in parallel.

---

## css-tree

**[css-tree](https://github.com/csstree/csstree)** is a tool set for working with CSS, including a fast parser, generator, and traversal utilities. It creates an abstract syntax tree (AST) of CSS for programmatic analysis and manipulation.

---

## ESLint

**[ESLint](https://eslint.org/)** is a static code analysis tool for identifying problematic patterns in JavaScript and TypeScript code. It helps enforce coding standards and catch potential errors before runtime.

---

## htmlparser2

**[htmlparser2](https://github.com/fb55/htmlparser2)** is a fast and forgiving HTML/XML parser. It's designed to handle real-world markup (including malformed HTML) and provides a streaming interface for parsing large documents efficiently.

---

## jsdom

**[jsdom](https://github.com/jsdom/jsdom)** is a JavaScript implementation of web standards, particularly the DOM and HTML standards. It allows you to run browser-based code in Node.js by simulating a browser environment without needing an actual browser.

---

## LangChain.js

**[LangChain.js](https://js.langchain.com/)** is a framework for developing applications powered by large language models (LLMs). It provides abstractions for prompt management, agent orchestration, tool integration, and working with multiple LLM providers.

---

## Prettier

**[Prettier](https://prettier.io/)** is an opinionated code formatter that enforces consistent code style across JavaScript, TypeScript, CSS, Markdown, and many other languages. It removes debates about formatting by automatically applying a standardized style.

---

## Vite

**[Vite](https://vitejs.dev/)** is a modern frontend build tool that provides lightning-fast development server startup and hot module replacement (HMR). It uses native ES modules during development and Rollup for optimized production builds.

---

## Zod

**[Zod](https://zod.dev/)** is a TypeScript-first schema declaration and validation library. It allows you to create schemas that automatically infer static TypeScript types, providing runtime validation with compile-time type safety.
