# Architecture Decision Records (ADR)

Short, dated notes that capture **why** a decision was made, not only what changed.

## When to add one

- Choosing a library, protocol, or deployment shape that is hard to reverse.
- Trade-offs that future contributors will question without context.

## Format

Use the template in [0001-record-architecture-decisions.md](./0001-record-architecture-decisions.md). File names: `NNNN-short-title.md` (increment `NNNN`).
